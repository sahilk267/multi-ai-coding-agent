Project memory — Multi-AI Autonomous Coding Agent
What this project actually is
A browser-based, no-API-key autonomous coding agent (Cursor/Devin/Windsurf-style) that drives the public web UIs of ChatGPT, DeepSeek, Qwen, and Gemini from a Chrome MV3 extension, while a local FastAPI backend on 127.0.0.1:8765 owns the filesystem, shell, tests, and git.

Everything lives under ai-agent/. The artifacts/ directory at the repo root contains unrelated leftover scaffold workflows (api-server, mockup-sandbox) — these are not part of this project and must not be modified.

Architecture (one-line summary)
Chrome MV3 extension (planner → router → 4 provider adapters → executor) ↔ WebSocket ↔ FastAPI backend (FS / shell / tests / git / watcher).

Full diagrams live in ai-agent/docs/architecture.md (component, sequence, state machine — Mermaid).

Routing policy (no-API mode, web-UI driven)
Task kind	Primary	Fallback order
planning	ChatGPT	DeepSeek → Gemini → Qwen
coding	DeepSeek	ChatGPT → Qwen → Gemini
debugging	Qwen	DeepSeek → ChatGPT → Gemini
long_context	Gemini	ChatGPT → DeepSeek → Qwen
Token budgets per provider live in extension/config.json modelLimits and are enforced by extension/core/tokenManager.js (budgetPrompt), wired into background.js sendPromptToProvider.

Key commands
All commands run from ai-agent/.

make install   # extension npm i + backend pip install -r requirements.txt
make run       # uvicorn backend.server:app --port 8765
make test      # extension unit tests (token budgeter: 10/10)
make check     # selector audit + config snapshot + token-budget CLI
make smoke     # backend smoke test (requires python3)
make verify    # test + check + smoke — the one-shot CI gate
make hooks     # install git pre-push hook that runs 'make verify' (one-time)
make unhooks   # uninstall the git pre-push hook
make clean     # remove __pycache__, node_modules, logs
Individual extension scripts:

npm run test — unit tests (extension/tests/)
npm run budget — print token budget for a sample prompt
npm run check-selectors — audit each adapter has the 4 required selector arrays
npm run snapshot-config — verify routing matrix is consistent and primaries are enabled
npm run load-check — static "would Chrome load this?" check (manifest + referenced files + config keys)
npm run package-check — release smoke: build zip → verify integrity → extract → load-check the extracted tree (skips if zip/unzip missing)
npm run check — runs all five above
Security model (enforced in backend/security.py)
Allow-list of binaries: npm, npx, node, yarn, pnpm, python(3), pip(3), pytest, uv, git, cargo, rustc, go, mvn, gradle, java, make, ls, cat, echo, pwd.
Blocked tokens: &&, ||, ;, |, >, <, `, $(, rm -rf, sudo, chmod 777, mkfs, :(){, curl , wget .
Blocked flags: --privileged, --no-preserve-root.
Path traversal: safe_resolve(workdir, rel_path) rejects any path resolving outside the active project root → HTTP 403.
Default timeout: 60 s for /execute and /run_tests.
Network: bound to 127.0.0.1:8765 only, no auth (intentional — local-only).
State machine + crash recovery
States: IDLE → PLANNING → CODING → TESTING → DEBUGGING → COMMITTING → DONE (with WAITING_APPROVAL interrupts).
Service worker checkpoints STATE to chrome.storage.local every 5 s.
Backend mirrors session memory to memory/session_memory.json.
On startup, the snapshot is restored — survives Chrome service-worker suspension and full browser restarts.
